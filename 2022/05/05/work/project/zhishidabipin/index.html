GAME_RECORD_USER_SET

  • 加入房间的用户列表

RANK

  • 排行榜
  • 竞答与抢答模式情况下
    • 结束按钮会广播到所有用户(包括主控端)
    • 主控端倒计时结束,自动结束答题情况下会广播到所有用户(包括主控端)
  • 竞答模式下
    • 某位用户答题结束后,会广播到已经答题完成的用户

      TIMER

  • 倒计时
  • 每秒推送一次
  • 抢答模式
    • 用于控制主控端答题索引和倒计时
  • 竞答模式
    • 用于控制主控端与用户端答题索引和倒计时

PAUSE

  • 开始

PAUSE

  • 暂停

GOT_QUICK

  • 获得答题权
  • 抢答模式
    • 收到此广播将有答题权利

      GOT_USER_MASTER_QUICK

  • 拥有答题权的用户广播到主控端
  • 抢答模式
    • 主控端可以收到
      • 哪位用户获取到了答题权

NOT_GET_QUICK

  • 未获取答题权
  • 抢答模式
    • 用户端可以收到
      • 收到此广播表示 自己没有答题权

RESULT_QUICK

  • 抢答结果
  • 抢答模式
    • 用户端 拥有答题权的用户会收到
    • 主控端会收到

后台设计理由

  1. 主题与题目只能由excel导入,不能通过文本框添加.
    • excel导入便捷,文本框添加题目几乎不会被用到
    • 尽可能精炼功能点,让洁面看的清爽,只保留刚需功能
    • 一线操作员对电脑掌握情况可能并不熟练 集中到excel上操作适用性更广

问题

  • 抢答题答错扣分 是
  • 预抢答时间内无人抢答如何处理, 进入下一题? 无需预抢答
  • 竞答模式下 共有10题与名用户 用户1答题9道、用户2答题5道、用户3答题8道.主控端主动结束答题 是
    • 问 排行榜是否为空
  • 建议只要用户参与游戏即使退出也会出现在排行榜 是
    • 服务端没办法准确的判断用户是退出程序还是未答题
    • 如果长连接必须在线才算排名,有些用户答完可能熄屏,就会判定为离线,此时将会没有排名
  • 竞答模式排行榜什么时候出现（1.控制端手动结束答题2.主控端倒计时结束）,移动端是否可以看到 不需要
  • 抢答模式下在倒计时最后几秒是否提示用户正确答案 不需要
  • 用户抢答以后,主控端是否需要显示谁在答题 不需要
  • 统一设置答题得分 因为规则说明是统一的 是
  • 抢答模式下的倒计时规则
    • 是否有等待抢答时间，例如等待三秒后才出现抢答按钮 不需要
    • 抢答如果长时间未答题是否会跳到下一题 是
  • 点击结束按钮会结束答题,如果最后一题时间用尽是否会自动结束答题 是
  • 抢答与竞答都可以暂停 是
  • 抢答模式答错是否需要扣分,如果扣分是否会有负分情况 暂时按照0分扣分
  • 抢答模式下耗时是否为恒定. 是同时竞答模式不是恒定
    • 主控端设置每题,用时10s,得分10分,共6题,2名用户参加
    • 用户1抢到4题,答对3题,错误1题
    • 用户2抢到2题,答对2题,错误0题
    • 结束后排行榜
    • 排名1 用户1(昵称头像) 分数? 用时?
    • 排名2 用户2(昵称头像) 分数? 用时?
  • 扫码后长时间未填写昵称与头像,直到主控端开始答题后才填写昵称与头像,是否可以加入答题 不允许
  • excel导入题目是插入还是覆盖 相同题目覆盖
  • 扫码进入页面有了三名用户进入,此时主控端返回到”显示题型分类”页面,用户端是什么效果 结束效果
  • 一场比赛时间是否恒定,暂停是否会影响用时.暂停会影响用时
